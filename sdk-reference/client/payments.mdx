## `useSubscription()`

```ts
function useSubscription(): {
  // State
  tier: string                          // current plan slug; 'free' if no plan
  status: 'none' | 'trialing' | 'active' | 'past_due' | 'canceled' | string
  entitled: boolean                     // true iff status is active or trialing
  interval: 'month' | 'year' | null
  currentPeriodEnd: string | null       // ISO timestamp
  cancelAtPeriodEnd: boolean
  trialEndsAt: string | null
  plans: Plan[]                         // catalog from src/subscriptions.ts
  isLoading: boolean

  // Predicates
  hasTier:   (slug: string) => boolean  // strict match + entitled
  isAtLeast: (slug: string) => boolean  // rank ≥ target + entitled

  // Actions (browser-redirect)
  subscribe:  (slug: string, opts?: { interval?: 'month' | 'year'; returnUrl?: string; cancelUrl?: string }) => void
  openPortal: (returnUrl?: string) => void

  // Refresh
  refresh: () => Promise<void>
}
```

<Warning>
**Gate features on `hasTier` / `isAtLeast`, never on `tier` alone.** A `past_due` Pro subscriber keeps `tier === 'pro'` but `entitled === false`.
</Warning>

## `useCheckout({ productId?, ... })`

```ts
function useCheckout(options?: { productId?: string }): {
  chargeOnce: (opts:
    | { productId: string; returnUrl?: string; cancelUrl?: string }
    | { amount: number; name: string; description?: string; returnUrl?: string; cancelUrl?: string }
  ) => void
  purchases: Purchase[]
  owned: boolean                           // requires productId on hook
  ownsProduct: (productId: string) => boolean
  refresh: () => Promise<void>
}
```

Two modes:

- **Product mode** — pass `{ productId }` to both the hook and `chargeOnce`. Entitlement is checked via `owned` / `ownsProduct`.
- **Ad-hoc mode** — pass `{ amount, name }` to `chargeOnce`. The purchase has `productId: null` and **cannot** be used to gate features.

## `<PricingTable plans={...} />`

A ready-made pricing UI wired to the plan catalog.

```tsx
import { PricingTable, useSubscription } from 'deepspace'

function Pricing() {
  const { plans } = useSubscription()
  return <PricingTable plans={plans} />
}
```

| Prop | Type | Description |
|---|---|---|
| `plans` | `Plan[]` | Plans from `useSubscription().plans` |
| `onCheckout` | `(slug: string, interval: 'month' \| 'year') => void` (optional) | Intercept clicks |
| `className` | `string` | Tailwind utilities applied to the outer container |

## Server helpers

### `requireSubscription(c, opts)`

Throws if the caller's subscription doesn't meet the requirement. Use inside Hono route handlers.

```ts
function requireSubscription(c: Context, opts:
  | { tier: string }      // strict slug match
  | { atLeast: string }   // rank ≥ target
): Promise<SubscriptionState>

class SubscriptionAuthError extends Error      // 401-shaped
class SubscriptionRequiredError extends Error  // 402-shaped, exposes .required
```

```ts
app.get('/api/premium', async (c) => {
  try {
    await requireSubscription(c, { atLeast: 'pro' })
  } catch (e) {
    if (e instanceof SubscriptionAuthError)     return c.json({ error: 'unauthenticated' }, 401)
    if (e instanceof SubscriptionRequiredError) return c.json({ error: 'upgrade_required', required: e.required }, 402)
    throw e
  }
  // protected logic
})
```

### `getSubscription(c)`

Read-only variant. Returns the subscription state without throwing.

```ts
function getSubscription(c: Context): Promise<SubscriptionState | null>
```

### `cancelSubscription(c, opts)`

Cancel one user or every user on a retired plan. Requires app owner JWT.

```ts
function cancelSubscription(c: Context, opts:
  | { userId: string; atPeriodEnd?: boolean }
  | { planSlug: string; atPeriodEnd?: boolean }
): Promise<{ canceled: number; hasMore: boolean }>

class CancelSubscriptionError extends Error
```

Batched at 50. Loop on `hasMore`.

### `refundInvoice(c, opts)`

Refund a charge by its local invoice ID (not the Stripe `inv_xxx` ID).

```ts
function refundInvoice(c: Context, opts: {
  invoiceId: string
  amount?: number               // cents; full refund if omitted
  reason?: 'requested_by_customer' | 'duplicate' | 'fraudulent'
}): Promise<{ refunded: number; refundId: string }>

class RefundError extends Error
```

Constraints: 90-day window, 50 per 24h per app, no overdraw.

## Plan manifest types

Declared in `src/subscriptions.ts` and `src/products.ts`:

```ts
type SubscriptionPlan = {
  slug: string
  name: string
  priceCents: number        // monthly; 0 = free tier
  yearlyCents?: number
  trialDays?: number        // max 90
  taxCode?: string
}

type OneTimeProduct = {
  productId: string         // stable entitlement key
  name: string
  amountCents: number       // min 100 ($1.00)
  description?: string
}
```
