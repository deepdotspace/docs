The messaging API ships as drop-in [schemas](/sdk-reference/worker/schemas) and React hooks. Add the schemas to your app's `RecordRoom`, mount the providers (already wired in the scaffold), and call the hooks.

```ts
import {
  useChannels, useMessages, useReactions,
  useChannelMembers, useReadReceipts, useConversation,
  formatMessageTime, formatFullTimestamp, shouldGroupMessages,
  getThreadCounts, groupReactionsForMessage, parseMessageMetadata,
  getConversationDisplayName, getConversationParticipantIds,
  isDMConversation,
} from 'deepspace'
```

Every messaging hook returns `status: 'loading' | 'ready' | 'error'` and `error?: string` alongside its records. Gate skeleton states on `status`.

For an end-to-end walkthrough including UI patterns, see the [messaging guide](/guides/messaging).

## `useChannels()`

```ts
function useChannels(): {
  channels: Channel[]
  status: 'loading' | 'ready' | 'error'
  error?: string
  create:  (input: { name: string; type: 'public' | 'private' | 'dm'; description?: string }) => Promise<string>
  update:  (channelId: string, patch: Partial<Channel>) => Promise<void>
  archive: (channelId: string) => Promise<void>
  remove:  (channelId: string) => Promise<void>
}
```

<Tabs>
  <Tab title="Create">
    `create({ name, type })` requires both fields. Passing only `{ name }` returns a channel with `type: undefined` and silently breaks downstream queries.

    ```tsx
    const { create } = useChannels()

    const channelId = await create({
      name: 'general',
      type: 'public',
      description: 'Company-wide announcements',
    })
    ```

    Channel types: `'public'` (any signed-in user), `'private'` (members only), `'dm'` (two-user direct message).
  </Tab>

  <Tab title="Update">
    `update` patches channel metadata — name, description, archived state, custom fields. Uses [merge semantics](/sdk-reference/client/records#usemutations-t-collection).

    ```tsx
    const { update } = useChannels()

    await update(channelId, { description: 'Updated description' })
    ```
  </Tab>

  <Tab title="Archive">
    Hides the channel from default lists without deleting messages. Reversible — set `archived: false` via `update` to restore.

    ```tsx
    const { archive } = useChannels()

    await archive(channelId)
    ```
  </Tab>

  <Tab title="Delete">
    Hard delete. The channel record and all associated messages remain in the DO until the messages are also deleted. Prefer `archive` for user-facing flows.

    ```tsx
    const { remove } = useChannels()

    await remove(channelId)
    ```
  </Tab>
</Tabs>

## `useMessages(channelId, options?)`

```ts
function useMessages(channelId: string, options?: {}): {
  messages: Message[]
  status: 'loading' | 'ready' | 'error'
  error?: string
  send:        (input: { content: string; metadata?: object }) => Promise<string>
  edit:        (messageId: string, patch: Partial<Message>) => Promise<void>
  softDelete:  (messageId: string) => Promise<void>
  remove:      (messageId: string) => Promise<void>
}
```

<Tabs>
  <Tab title="Send">
    Posts a new message to the channel. Returns the new `messageId`. Identity (author) is derived from the [verified JWT](/concepts/architecture#security-model--websocket-identity) — you don't pass an `authorId`.

    ```tsx
    const { send } = useMessages(channelId)

    const messageId = await send({
      content: 'Hello, world',
      metadata: { mentions: ['user_abc'] },
    })
    ```
  </Tab>

  <Tab title="Edit">
    Patches an existing message. The collection's RBAC determines who can edit; the default `MESSAGES_SCHEMA` allows authors to edit their own.

    ```tsx
    const { edit } = useMessages(channelId)

    await edit(messageId, { content: 'Updated text' })
    ```
  </Tab>

  <Tab title="Soft delete">
    Sets a tombstone flag instead of removing the row. **Preferred over `remove`** — preserves chat history continuity and lets the UI render a "[deleted]" placeholder.

    ```tsx
    const { softDelete } = useMessages(channelId)

    await softDelete(messageId)
    ```
  </Tab>

  <Tab title="Hard delete">
    Permanently removes the row. Use sparingly — soft delete is almost always the right choice for chat history.

    ```tsx
    const { remove } = useMessages(channelId)

    await remove(messageId)
    ```
  </Tab>
</Tabs>

## `useReactions(channelId)`

```ts
function useReactions(channelId: string): {
  reactions: Reaction[]
  status: 'loading' | 'ready' | 'error'
  error?: string
  getReactionsForMessage: (messageId: string) => Reaction[]
  toggle: (messageId: string, emoji: string) => Promise<void>
}
```

`getReactionsForMessage` is an O(1) lookup. `toggle` adds the caller's reaction or removes it if it already exists.

```tsx
const { getReactionsForMessage, toggle } = useReactions(channelId)

<button onClick={() => toggle(messageId, '👍')}>
  👍 {getReactionsForMessage(messageId).filter(r => r.data.emoji === '👍').length}
</button>
```

## `useConversation(options?)`

For DM/conversation Durable Objects (scope `conv:<id>`) backed by `conv_messages` / `conv_reactions` / `conv_members` collections. Mount inside a `<RecordScope roomId="conv:..." schemas={CONVERSATION_SCHEMAS}>`.

```ts
function useConversation(options?: {}): {
  messages: ConvMessage[]
  reactions: ConvReaction[]
  members: ConvMember[]
  status: 'connecting' | 'connected'
  send:           (input: { content: string }) => Promise<string>
  edit:           (messageId: string, patch: object) => Promise<void>
  remove:         (messageId: string) => Promise<void>
  toggleReaction: (messageId: string, emoji: string) => Promise<void>
}
```

Different from `useMessages` / `useReactions` / `useChannelMembers` — those target the channel-style collections within your app's main `RecordRoom`. `useConversation` targets a dedicated DM Durable Object on a `conv:<id>` [scope](/concepts/architecture#scopes).

## Record types

```ts
type Channel = Envelope<{
  name: string
  type: 'public' | 'private' | 'dm'
  description?: string
  archived?: boolean
}>

type Message = Envelope<{
  channelId: string
  authorId: string
  content: string
  metadata?: object
  deleted?: boolean
}>

type Reaction = Envelope<{
  messageId: string
  authorId: string
  emoji: string
}>

type ChannelMember = Envelope<{
  channelId: string
  userId: string
  joinedAt: string
}>

type ReadReceipt = Envelope<{
  channelId: string
  userId: string
  lastReadMessageId: string
}>
```

The [envelope](/concepts/data-model#collections-and-records) shape (`recordId`, `data`, `createdBy`, `createdAt`, `updatedAt`) wraps every record.

## Helper functions

| Helper | Signature |
|---|---|
| `formatMessageTime(dateStr)` | Returns `'3:42 PM'` |
| `formatFullTimestamp(dateStr)` | Returns `'Today at 3:42 PM'` |
| `shouldGroupMessages(current, previous, options?)` | True if consecutive messages from the same author within a window |
| `getThreadCounts(messages)` | Map of parent messageId → reply count |
| `groupReactionsForMessage(reactions, messageId, currentUserId)` | Aggregates reactions by emoji |
| `parseMessageMetadata(raw)` | Safe JSON parse of `metadata` field |

## Conversation helpers

| Helper | Signature |
|---|---|
| `getConversationDisplayName(conv)` | Resolves a display string from a conversation record |
| `getConversationParticipantIds(conv)` | Returns the array of participant `userId`s |
| `isDMConversation(type)` | True if `type === 'dm'` |

## Directory conversations

For cross-app conversations (the platform's shared `dir:<appHandle>` DO), use the directory hooks instead of `useChannels` / `useMessages`. These hooks support discoverable conversations, communities, and posts spanning multiple DeepSpace apps. See the [worker schemas reference](/sdk-reference/worker/schemas#drop-in-schemas--directory-cross-app) for `DIRECTORY_SCHEMAS`.
